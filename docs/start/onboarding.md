# Onboarding (App)

This doc describes the **planned** first-run onboarding flow for a companion app (not yet available). The goal is a smooth “day 0” experience: pick where the Gateway runs, connect auth, run the wizard, and let the agent bootstrap itself.

For the current onboarding path, see [Onboarding Overview](/start/onboarding-overview) and [CLI Wizard](/start/wizard).
